Introduction

The API is used to calculate the visitor count of website and pagwview count of the website which is using a local database in form of an array stored in form of count.json.
The authentication system is designed for signIn and Signup with error handling.
The authentication system uses json web token and hashing of password with salt to provide reliable security.
The database used is MongoDB
The API is live on deployment server i.e. Deta.sh : https://8ejo5c.deta.dev/

Installation Locally

In order to run the API locally clone the repository from my github account : https://github.com/Ani-Bos/Microtask2.git

Then cd second_micro

Then install the dependencies as required for this microtask like cors , express , dotenv , jsonwebtoken , mongodb , mongoose ,bcrypt and nodemon like npm i express cors dotenv jsonwebtoken mongodb mongoose bcrypt nodemon.

Then run " npm run dev "

In order to fetch the visitor count and pageview and to get a response from get request use "/visito/api" path .

In order to see a much interactive way to find visitor and pageview count run index.html and can observed the dynamic change in counts in the website.

About repository

Visitor count

This repository is created for building an API for MicroTask 2 given by HostelDevta in developement phase.I have created an express server and made an array having count of pageview and visitor count which is dynamically updated by a function written in visitor.js file. So when a visitor first run the website pageview==1 && visit==1 but when he/she refresh the page pageview increases accordingly but visit remain same but if open the URL in new browser then view==2 and increases dynamically and pageview also gets updated.

Authentication

For Authentication I have created two checkpoints i.e either the user is new i.e. he/she have to signup or the user have already signup he/she have to sigin.After login successfully since I have connected it to mongodb as it is my database it will generate the { username , password , email } as the result also there will be a token which can be used for checking authentication and can be used for several other purpose in which token is required.
